# AWS Live Streaming Pipeline via Terraform

## Overview

This project deploys a complete AWS live streaming workflow using Terraform.

Architecture:

FFmpeg-SRT-Caller -> MediaConnect-SRT-Listener -> MediaLive -> MediaPackage V1 -> HLS(.m3u8)/DASH(.mpd) ABR Endpoints

## Features

- Infrastructure as Code (Terraform)
- AWS MediaConnect
- AWS MediaLive
- AWS MediaPackage V1
- HLS ABR Delivery
- DASH ABR Delivery
- CloudFormation integration for MediaPackage Origin Endpoints

## Current Working Status 

- MediaConnect Flow
- MediaLive Input
- MediaLive Channel
- MediaPackage Channel
- HLS ABR Playback
- DASH ABR Playback

## Demo Workflow

This Terraform project provisions the AWS Media Services part of the live streaming pipeline.

The infrastructure created by this project includes:

* AWS MediaConnect SRT listener flow
* AWS MediaLive input connected to MediaConnect
* AWS MediaLive channel for ABR/transcoding workflow
* AWS MediaPackage v1 channel
* MediaPackage HLS endpoint
* MediaPackage DASH endpoint
* Required IAM roles and permissions

### Clean Demo Preparation

For a clean live demonstration, the existing lab infrastructure can be removed first:

```powershell
terraform destroy
```

Before running `terraform destroy`, make sure:

* The MediaLive channel is stopped.
* The FFmpeg SRT caller source is stopped.
* No active test stream is running.

After the destroy completes, the AWS Console should show that the previous lab MediaConnect, MediaLive, and MediaPackage resources have been removed.

### Provision AWS Media Pipeline

Run:

```powershell
terraform init
terraform validate
terraform plan
terraform apply
```

After successful deployment, Terraform outputs the runtime values needed by the EC2 FFmpeg source project:

```text
source_ingest_ip
source_ingest_port
hls_endpoint_url
dash_endpoint_url
medialive_channel_id
medialive_input_id
mediapackage_channel_id
flow_name
flow_arn
```

Example usage:

```text
source_ingest_ip + source_ingest_port
```

are used by the EC2 FFmpeg SRT caller as the SRT destination.

```text
hls_endpoint_url
dash_endpoint_url
```

are the final playback outputs generated by AWS MediaPackage.

### Runtime Integration

After Terraform completes, copy these values into the EC2 runtime project `.env` file:

```text
SRT_TARGET_IP=<terraform source_ingest_ip>
SRT_TARGET_PORT=<terraform source_ingest_port>
HLS_URL=<terraform hls_endpoint_url>
DASH_URL=<terraform dash_endpoint_url>
```

The EC2 runtime project then starts FFmpeg in SRT caller mode and sends the video loop into the Terraform-provisioned AWS MediaConnect listener.

### Cleanup

To avoid ongoing AWS Media Services cost, stop the MediaLive channel and then run:

```powershell
terraform destroy
```

This removes the Terraform-managed AWS Media Services resources.
